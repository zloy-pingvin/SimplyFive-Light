SimplyFive · User Manual

SimplyFive — One-click LOD Generator

A Blender add-on that generates a set of LOD meshes (Levels of Detail) from the active object, using the meshoptimizer library. For each mesh you can create up to 5 simplified versions with individual aggressiveness; simplification can take UV unwrapping, normals, vertex colors (as an importance map) and materials into account.

1.What the add-on does

SimplyFive creates several simplified copies of your model in one click (LOD — Level of Detail): the farther an object is from the camera in a game or scene, the fewer polygons it needs. The add-on figures out on its own how to remove the “extra” triangles so that the silhouette, textures and shading suffer as little as possible.

2.Installing the add-on

  1. Open Blender → Edit → Preferences → Add-ons.
  2. Click “Install...” and select the SimplyFive folder/archive.
  3. Enable the checkbox next to the add-on in the list.

2.1 The meshoptimizer library

The add-on uses the third-party meshoptimizer library, which does the actual geometry simplification. It is already bundled inside the add-on — no extra steps are needed, the generation buttons work right after installation. You can open the add-on’s Preferences to confirm the library status reads ready.

Pro Tip: Pro can also rebuild the library from source (e.g. to pull a newer meshoptimizer) with a button in Preferences — see Appendix A. Light always ships and loads the bundled library only.

3.Where to find the add-on panel

  1. Select the mesh object you want to simplify in the 3D viewport.
  2. Open the sidebar with the N key.
  3. Find the “LODs” tab on the right (Pro the Pro tab is “LODs”; the Light tab is “LODsLight”).

At the top of the panel you’ll see the selected object’s name and its current triangle / vertex count.

4.Creating LODs in 5 steps

  1. Select the model (best to work with final, finished geometry).
  2. In “Number of LODs”, set how many levels you need (3–4 is usually enough).
  3. For each level, pick a Mode from the dropdown: the farther the level is from “0”, the more aggressive the mode should be (see the table in section 5).
  4. Optionally adjust the triangle percentage (%) for each level.
  5. Click the big “Generate LODs” button at the bottom.

After generation, new objects appear: your original is renamed to name_lod_0, and name_lod_1, name_lod_2, … appear next to it — one per level.

Important: Running generation again for the same object deletes and recreates old LOD objects with the same names — previous versions are not kept.

5.Which mode to choose for which level

Each LOD level has a “Mode” dropdown with ready-made settings. You don’t need to understand every parameter — just go by how far this level will be from the camera:

Level (by distance)ModeWhen to use it
Closest (LOD1)CarefulNearly identical to the original — for close-ups / short distances.
Medium (LOD2–3)StandardA good balance of quality and polygon count — the “workhorse”.
Far (LOD4)AggressiveNoticeably fewer polygons; slight texture distortion is acceptable.
Very far (LOD5)Very Aggressive / Clear SimplifySeen small or from afar — only the silhouette shape matters.
Tip: If the model looks “broken” after simplification (holes, textures gone haywire) — switch to a gentler mode for that level, or raise the percentage (bring % closer to 100).

Pro For finer control, expand “Details” under the mode — that’s where all the individual parameters live (section 7). In Light the modes are fixed presets and the Details block is not editable.

6.General options (below the level list)

These apply to the whole generation, not to a single level.

OptionWhat it does and when to enable it
Error Absolute
(for multiple materials)		 Switches the meaning of Target Error from “a fraction of the model’s size” to “an absolute distance”. Especially useful when the model has several materials: without it, parts with different materials can simplify unevenly. Turn it on if you notice that imbalance.
Merge by Distance Welds together vertices that ended up at exactly the same point after simplification (Blender’s own tool). Keep this on — it cleans up the result without harming textures / seams.
Merge Threshold The welding threshold above. Keep it small (the default is usually fine) — a large value can weld together thin parts meant to stay separate.
Multiple UV Channels Carries every UV channel of the source onto the LODs (not just the active one), keeping layer names and active/render flags. Off = only the active UV channel is kept. On = geometric quality can drop slightly at the same %, because extra UV seams (e.g. lightmap islands in UV2) constrain simplification.

6.1 Vertex colors as an “importance map”

If important areas are painted white (in Vertex Paint mode) and unimportant ones black, the add-on can treat important zones (a face, a logo, a small detail) more carefully and simplify unimportant ones (a back, a sole) more aggressively.

OptionWhat it does
Use Vertex Color as Importance Turns the idea on. Reads the active vertex color layer as a per-vertex importance map (luminance: white = important). Applies to every LOD.
Importance Strength How strongly the painting biases the result (0–1). A soft weight: higher = more careful around white zones, but not a full guarantee even at maximum.
Hard-Lock Above Threshold Pro For an actual guarantee: vertices whose importance is above the threshold below are fully protected from collapsing (same hard-lock mechanism as Protect UV Seams).
Lock Threshold Pro The brightness threshold (0–1) for the hard protection above. E.g. 0.9 = only the brightest painted zones get hard-protected.

7.Fine-tuning each level — the “Details” section Pro

The built-in modes are enough for most cases. But if something’s off — the model “breaks”, the texture drifts, or too many polygons remain — expand “Details” under that level’s mode and adjust the parameters by hand. Changing any of them switches the mode to “Custom” automatically.

7.1 Target Error — the main parameter

The “allowed error” of simplification — how much the result may differ from the original (by default a fraction of the model’s size, 0–1).

  • Lower → stops earlier and more cautiously; closer to the original, but more triangles than the target % may remain.
  • Higher → allowed to cut harder to reach the target %, at the cost of more visible distortion.
Important: Too high a Target Error can collapse the mesh completely (especially with a low %). If the object caves in or disappears — lower Target Error first.

7.2 Shape and mesh boundaries

ParameterWhat it does
Lock Open EdgesPrevents open/boundary edges from being touched. Keep on if the model has open edges you don’t want deformed.
SparseHelps when the model is several disconnected pieces combined into one object.
Prune (aggressive)Lets the add-on fully discard small “cheap” disconnected bits. Use when a level won’t drop to the target count due to debris.
RegularizeMakes result triangles more uniform, at a slight cost to silhouette accuracy. Useful for meshes that get deformed later (rigging, sim).

7.3 Textures (UV) and shading (normals)

ParameterWhat it does
Permissive (aggressive)Lets simplification jump over texture seams if it greatly cuts polygons. For far, rough LODs; not for close-ups.
Protect UV SeamsOnly with Permissive: protects seams where the texture clearly differs on either side, cutting “invisible” seams freely.
Preserve UVs & NormalsThe main “smart” toggle: tries not to ruin texture and shading. Turn off only for the farthest/roughest LODs.
Normal WeightHow strongly shading is protected (0–1). Increase if you see creases/odd highlights; decrease to push polygons lower.
UV WeightSame idea for texture coordinates. Increase if the texture drifts/stretches; decrease for far LODs.
Vertex UpdateSlightly moves vertices and texture to fit the new shape. Better results at aggressive levels (LOD3+).

7.4 Special modes for the farthest LODs

ParameterWhat it does
Pre-pruneA separate pass BEFORE main simplification (threshold 0–0.2) that strips small debris without affecting Target Error accuracy. Start at 0.01–0.05.
Sloppy (topology-ignoring)A rough mode for the farthest levels: ignores shape and seams entirely and cuts as much as possible. Vertex Update and the weights have no effect when on.

8.Checking the result

After generation, every level sits stacked in the scene. To evaluate a specific level:

  • Move the “LOD Preview (distance)” slider at the top — 0 shows the original, higher values show the corresponding simplified level (others hide automatically).
  • Or click “Only This LOD” next to the level you want; “Show All LODs” makes every level visible again.
  • The “Line Up LODs” button lays every level out in a row (isolated in local view) to compare the progression side by side.

9.Fixing just one level

You don’t have to recreate everything. Change the settings for the level you want and click “Generate This LOD” in its block — the other levels stay untouched.

10.Saving your own settings Pro

If you’ve worked out a configuration you like (number of levels, percentages, modes, and every Details value), save it as a preset and apply it to other models in one click:

  1. Set up all the LOD levels the way you want.
  2. Click the “+” next to the “LOD Presets” dropdown and name the preset.
  3. Next time, just pick this preset from the list — every setting is applied instantly.

The “Restore Defaults” button in the same menu resets settings to factory values.

11.Frequently asked questions

Building the library fails ProCheck that a C/C++ compiler is installed and you have internet access. Full error text is in Blender’s system console.

Appendix A. Building a fresh meshoptimizer Pro

The library is already bundled, so this isn’t needed for normal use. Rebuild only if you want the very latest meshoptimizer from GitHub, or the bundled library is damaged.

Important: Building requires internet access and a C/C++ compiler installed (the same one used to build other C add-ons).
  1. Open Edit → Preferences → Add-ons → SimplyFive and expand its settings.
  2. Click “Build meshoptimizer (one-time)” (or “Rebuild”).
  3. Wait ~a minute — the add-on downloads the source, compiles it, and loads the library.
  4. Confirm the status changed to the “ready / built” state.

If something goes wrong, the error text is shown there and in Blender’s system console (Window → Toggle System Console on Windows) — usually a missing compiler or no internet.
